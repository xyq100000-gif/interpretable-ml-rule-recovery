# Portfolio Positioning Notes

## Recommended repository title

**From Classification to Rule Recovery: An Interpretable ML Pipeline**

Alternative titles:

1. **Recovering Hidden Rules in Synthetic Tabular Data**
2. **Interpretable Structure Discovery from Synthetic Competition Data**
3. **Sparse Rule Discovery in a Synthetic ML System**

## How to describe the project in one sentence

A compact interpretable machine learning case study that reconstructs hidden categorical, ordinal, and linear decision rules from synthetic tabular data using sparse feature selection, threshold search, and formula recovery.

## What to emphasise

- model transparency rather than black-box fitting;
- sequential recovery of latent structure;
- sparse modelling choices;
- method decomposition and careful evaluation;
- explicit awareness of limits in a synthetic setting.

## What not to emphasise

- the original coursework narrative or fictional world-building;
- the tournament simulation as a central contribution;
- inflated novelty claims;
- generic statements such as “used machine learning to solve a complex problem”.

## Best use in an application package

This project should be used as:

- a supporting GitHub repository;
- a technical discussion example in interviews;
- evidence of interpretable modelling interest;
- a secondary CV project rather than the lead project.
